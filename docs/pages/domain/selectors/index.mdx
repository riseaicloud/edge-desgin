# Selectors

Domain selectors for choosing Edge Platform resources. All selectors support search, loading states, and optional "All" options.

## Import

```tsx
import {
  ClusterSelector,
  WorkspaceSelector,
  NamespaceSelector,
  NodeGroupSelector,
} from '@riseaicloud/components'
```

## Common Features

All selectors share these features:

- **Search**: Real-time filtering by resource name (ClusterSelector also filters by alias)
- **Loading State**: Shows loading indicator while fetching data
- **Clearable**: Users can clear the current selection
- **"All" Option**: Optional "All Resources" choice for filtering scenarios
- **Width Control**: Fixed width or responsive sizing

## ClusterSelector

Select clusters with support for alias names.

<Preview className="min-h-[260px] items-start">
  <ClusterSelector
    value="hw101"
    onValueChange={(value) => console.log('Selected:', value)}
    clusters={[
      { metadata: { name: 'hw101', annotations: { 'theriseunion.io/alias-name': 'Hardware Cluster 101' } } },
      { metadata: { name: 'hk-test', annotations: { 'theriseunion.io/alias-name': 'HK Test Cluster' } } },
      { metadata: { name: 'staging-01' } },
    ]}
    includeAllOption={true}
    placeholder="选择集群"
    width={300}
  />
</Preview>

```tsx
import { ClusterSelector } from '@riseaicloud/components'

function ClusterFilter() {
  const [selectedCluster, setSelectedCluster] = useState('all')
  const [clusters, setClusters] = useState<ClusterData[]>([])
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    // Fetch clusters from API
    fetchClusters().then(data => {
      setClusters(data)
      setLoading(false)
    })
  }, [])

  return (
    <ClusterSelector
      value={selectedCluster}
      onValueChange={setSelectedCluster}
      clusters={clusters}
      loading={loading}
      includeAllOption={true}
      placeholder="选择集群"
      width={300}
    />
  )
}
```

### ClusterSelector Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | Currently selected cluster name |
| `onValueChange` | `(value: string) => void` | — | Callback when selection changes |
| `clusters` | `ClusterData[]` | — | Cluster data array |
| `loading` | `boolean` | `false` | Show loading indicator |
| `placeholder` | `string` | `"选择集群"` | Placeholder text |
| `className` | `string` | — | Additional CSS classes |
| `width` | `string \| number` | — | Component width |
| `includeAllOption` | `boolean` | `true` | Include "所有集群" option |

**ClusterData Type:**

```tsx
interface ClusterData {
  metadata: {
    name: string
    annotations?: {
      'theriseunion.io/alias-name'?: string
    }
  }
}
```

## Edge Console Usage

These selectors are used throughout Edge Console for resource filtering and selection:

- **Cluster View**: Filter by cluster across multiple workspaces
- **Workspace View**: Switch between workspaces and filter resources
- **Namespace View**: Select namespace for workload deployment
- **Edge Management**: Choose node groups for edge device operations
