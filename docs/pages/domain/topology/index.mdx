# Topology

Cloud-edge collaboration topology visualization built on [@xyflow/react](https://reactflow.dev/). Renders the full hierarchy from host cluster down to edge nodes.

## Import

```tsx
import { ClusterTopology } from '@riseaicloud/components'
import type { TopologyNode } from '@riseaicloud/components'
```

---

## ClusterTopology

Interactive topology graph with auto-layout, collapse/expand, pan and zoom. The entire hierarchy is expressed as a recursive `TopologyNode` tree.

```tsx
import { ClusterTopology } from '@riseaicloud/components'
import type { TopologyNode } from '@riseaicloud/components'

const topologyData: TopologyNode = {
  id: 'host-cluster',
  name: 'hw101',
  type: 'hostCluster',
  status: 'healthy',
  metadata: { alias: 'Hardware Cluster 101', version: 'v1.28.4' },
  children: [
    {
      id: 'vc-01',
      name: 'vc-production',
      type: 'virtualCluster',
      status: 'healthy',
      children: [
        {
          id: 'ng-factory',
          name: 'ng-factory',
          type: 'nodeGroup',
          children: [
            { id: 'node-01', name: 'edge-node-01', type: 'node', status: 'healthy' },
            { id: 'node-02', name: 'edge-node-02', type: 'node', status: 'warning' },
          ],
        },
      ],
    },
  ],
}

function TopologyView() {
  return (
    <div style={{ height: 500 }}>
      <ClusterTopology
        data={topologyData}
        onNodeClick={(nodeId, nodeType) => {
          console.log('Clicked:', nodeId, nodeType)
        }}
      />
    </div>
  )
}
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `data` | `TopologyNode` | — | Root topology node (recursive tree) |
| `onNodeClick` | `(nodeId: string, nodeType: string) => void` | — | Node click callback |
| `className` | `string` | `''` | Additional CSS classes |

---

## Node Hierarchy

The topology uses a 5-layer model:

| Layer | Type | Domain | Description |
|-------|------|--------|-------------|
| L0 | `hostCluster` | Cloud | Root — the physical host cluster |
| L1 | `virtualCluster` | Cloud | vCluster hosted on the host cluster |
| L1 | `memberCluster` | Cloud | Remote member cluster joined to the platform |
| L2 | `memberVirtualCluster` | Cloud | vCluster hosted on a member cluster |
| L3 | `nodeGroup` | Edge | Group of edge nodes |
| L4 | `node` | Edge | Individual edge device/node |

---

## TopologyNode Type

```ts
interface TopologyNode {
  /** Unique ID for the node */
  id: string
  /** Display name */
  name: string
  /** Node type in hierarchy */
  type: NodeType
  /** Optional status indicator */
  status?: 'healthy' | 'warning' | 'error'
  /** Optional metadata (alias, version, nodeCount, ip, etc.) */
  metadata?: Record<string, string | number | undefined>
  /** Child nodes */
  children?: TopologyNode[]
}
```

---

## Edge Styles

Edges are automatically inferred from the parent–child relationship:

| Edge Type | From | To | Style |
|-----------|------|----|-------|
| `control` | Host cluster | Member cluster | Dark solid line |
| `virtual` | Any cluster | Virtual cluster | Dark dashed line |
| `edge` | Cloud → Edge | NodeGroup / Node | Green solid line |

---

## Layout

The topology uses an automatic hierarchical layout (Dagre):

- Cloud nodes (L0–L2) are arranged top-to-bottom with consistent horizontal spacing
- Edge nodes (L3–L4) extend below their parent cloud cluster
- Nodes can be collapsed/expanded by clicking the toggle button
- The canvas auto-fits on initial render and after layout changes

## Node Click Navigation

```tsx
<ClusterTopology
  data={topologyData}
  onNodeClick={(nodeId, nodeType) => {
    if (nodeType === 'nodeGroup') {
      router.push(`/nodegroups/${nodeId}`)
    } else if (nodeType === 'node') {
      router.push(`/nodes/${nodeId}`)
    } else {
      // cluster types
      router.push(`/clusters/${nodeId}`)
    }
  }}
/>
```
