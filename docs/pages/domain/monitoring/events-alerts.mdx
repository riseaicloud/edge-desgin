# Events & Alerts

Components for displaying Kubernetes events and firing alerts.

## EventsTable

K8s-aware events table from `@riseaicloud/components`. Accepts event data through props — no direct API calls.

### Import

```tsx
import { EventsTable } from '@riseaicloud/components'
import type { EventInfo } from '@riseaicloud/components'
```

### Preview

<Preview center={false}>

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `events` | `EventInfo[]` | — | Events array |
| `loading` | `boolean` | `false` | Show loading overlay |
| `error` | `string \| null` | `null` | Error message |
| `emptyMessage` | `string` | `"暂无事件"` | Empty state title |
| `emptyDescription` | `string` | `"该资源暂未产生任何事件"` | Empty state description |
| `className` | `string` | — | Additional CSS classes |

**EventInfo Type:**

```tsx
interface EventInfo {
  id?: number | string
  type: string        // 'Normal' | 'Warning' | 'Error'
  reason: string
  message: string
  timestamp: string   // ISO date string
  count: number
  source: string
}
```

## EventsViewer

Layer 3 wrapper around `EventsTable` from `@riseaicloud/components`. Provides the same interface.

### Import

```tsx
import { EventsViewer } from '@riseaicloud/components'
```

### Usage

```tsx
import { EventsViewer } from '@riseaicloud/components'

function PodEventsTab({ events, loading, error }) {
  return (
    <EventsViewer
      events={events}
      loading={loading}
      error={error}
      emptyMessage="该 Pod 暂无事件"
    />
  )
}
```

### Props

Same as `EventsTable` — see above.

## AlertList

A list of firing alerts from `@riseaicloud/components`. Accepts pre-fetched alert data.

### Import

```tsx
import { AlertList } from '@riseaicloud/components'
import type { AlertItem } from '@riseaicloud/components'
```

  <div className="p-4 w-full">
    <AlertList alerts={[]} />

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `alerts` | `AlertItem[]` | — | Alert list |
| `loading` | `boolean` | `false` | Show loading state |
| `error` | `string \| null` | `null` | Error message |
| `className` | `string` | — | Additional CSS classes |

**AlertItem Type:**

```tsx
interface AlertItem {
  id: string
  title: string
  severity: 'critical' | 'warning' | 'info'
  status: 'firing' | 'resolved'
  description?: string
  duration: string
  severityText: string
  labels: {
    namespace?: string
    pod?: string
    [key: string]: string | undefined
  }
}
```
