# Installation

## Prerequisites

- Node.js 18+
- pnpm 8+

## Install Packages

```bash
# Core tokens + UI components
pnpm add @riseaicloud/tokens @riseaicloud/ui

# Domain components (optional)
pnpm add @riseaicloud/edge-components

# Hooks (optional)
pnpm add @riseaicloud/hooks
```

## Configure Tailwind CSS

Add the Edge Design Tailwind preset to your `tailwind.config.js`:

```js
// @riseaicloud/tokens is an ESM package, so CJS `require()` returns the module
// namespace ({ default, edgePreset }) — the preset itself lives on `.default`.
// Hand the namespace object straight to `presets` and Tailwind reads
// `preset.theme` as `undefined`: the preset silently does nothing.
const presetModule = require('@riseaicloud/tokens/tailwind-preset')
const edgePreset = presetModule.default ?? presetModule

module.exports = {
  presets: [edgePreset],
  content: [
    './src/**/*.{ts,tsx}',
    './node_modules/@riseaicloud/ui/dist/**/*.{js,mjs}',
  ],
}
```

<Callout type="warning">
  **Don't skip the `.default`.** Without it there is no error — the preset is simply ignored,
  and every semantic utility (`bg-card`, `text-muted-foreground`, …) plus the whole `.dark`
  block disappears from your CSS. The failure looks like "the design system doesn't work"
  rather than a crash.
</Callout>

If your config is ESM (`tailwind.config.mjs` / `.ts`), import it directly — no `.default` dance:

```js
import edgePreset from '@riseaicloud/tokens/tailwind-preset'

export default {
  presets: [edgePreset],
  content: [/* … */],
}
```

The preset automatically:
- Injects all CSS custom properties (`:root` and `.dark`)
- Configures semantic color utilities (`bg-primary`, `text-muted-foreground`, etc.)
- Sets up border radius, shadows, z-index, and animation tokens
- Adds base styles for `body` and border colors

## Styles

**No stylesheet import is needed.** The package ships Tailwind class names, not a
compiled stylesheet — your own Tailwind build generates the CSS. Two things make
that work, both configured above:

1. **`presets: [edgePreset]`** — injects every CSS custom property (`:root` / `.dark`)
   and maps semantic utilities to `hsl(var(--token))`.
2. **`content` includes `./node_modules/@riseaicloud/ui/dist/**`** — without this the
   component class names are never scanned and their CSS is silently dropped.

<Callout type="warning">
  Skipping the `content` entry usually *appears* to work: any class your own code also
  uses gets generated anyway. The ones only the library uses (`bg-card`,
  `hover:bg-primary/80`, …) go missing — the failure is partial and easy to miss.
</Callout>

<Callout type="info">
  **Dark mode:** the `.dark` block is only emitted when the string `dark` appears
  somewhere in your scanned `content`. If you toggle the class purely from JS and never
  write a `dark:` utility in source, add `safelist: ['dark']` to your Tailwind config.
</Callout>

## Verify

```tsx
import { Button } from '@riseaicloud/ui'

export default function App() {
  return <Button variant="default">It works</Button>
}
```
