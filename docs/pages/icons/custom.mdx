# Custom Icons

当 Lucide React 图标库无法满足需求时，可以使用内联 SVG 或自定义 SVG 组件。

## 使用内联 SVG

最简单的方式是直接使用内联 SVG，通过 `className` 传入尺寸和颜色：

```tsx
function KubernetesIcon({ className }: { className?: string }) {
  return (
    <svg
      className={className}
      viewBox="0 0 24 24"
      fill="currentColor"
      xmlns="http://www.w3.org/2000/svg"
    >
      <path d="M12 2L2 7l10 5 10-5-10-5zM2 17l10 5 10-5M2 12l10 5 10-5" />
    </svg>
  )
}

// 使用
<KubernetesIcon className="h-5 w-5 text-blue-600" />
```

## 与 Lucide 保持一致

自定义图标应遵循与 Lucide 相同的接口，便于互换：

```tsx
import { type LucideProps } from 'lucide-react'

function EdgeIcon({ className, size = 24, ...props }: LucideProps) {
  return (
    <svg
      className={className}
      width={size}
      height={size}
      viewBox="0 0 24 24"
      fill="none"
      stroke="currentColor"
      strokeWidth={2}
      strokeLinecap="round"
      strokeLinejoin="round"
      {...props}
    >
      {/* SVG path */}
    </svg>
  )
}
```

## SVG 优化建议

1. **使用 `viewBox`**：确保 SVG 可缩放，通常设为 `0 0 24 24`
2. **使用 `currentColor`**：让图标继承文字颜色，与 Lucide 保持一致
3. **删除硬编码的 `width`/`height`**：改用 `className` 控制尺寸
4. **删除不必要的属性**：如 `xmlns:xlink`、`version` 等
5. **简化路径**：使用 [SVGO](https://svgo.dev) 工具优化 SVG 文件大小

## 图标来源推荐

| 来源 | 说明 |
|------|------|
| [Lucide](https://lucide.dev) | 首选，Edge Design 默认图标库 |
| [Simple Icons](https://simpleicons.org) | 品牌/技术图标（Kubernetes, Docker, GitHub 等） |
| [Heroicons](https://heroicons.com) | 备选，风格与 Lucide 接近 |

## 注意事项

- 优先使用 Lucide React 中已有的图标，避免引入额外的图标库
- 品牌图标（如 Kubernetes、Docker）使用官方 SVG，注意版权说明
- 不要在 `@riseaicloud/ui` 中添加业务专属图标，保持通用性
