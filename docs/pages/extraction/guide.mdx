# 组件提取指南

从 Edge Console 提取组件到 @riseaicloud/ui 的规范和流程。

## 提取边界

### ✅ 可以进 design 仓库的组件

1. **纯 UI 组件**
   - Button、Dialog、Badge 等基础组件
   - 不依赖业务逻辑的无状态组件

2. **通用模式组件**
   - LabelEditor（键值对编辑）
   - ConfirmDeleteDialog（删除确认）
   - EmptyState（空状态）
   - 不依赖 edge-specific 数据的类型

3. **工具函数和 Hooks**
   - `cn` - 类名合并工具
   - `useWizard` - 向导状态管理 Hook

### ❌ 禁止进入的组件

1. **业务选择器**
   - ❌ ClusterSelector
   - ❌ NamespaceSelector
   - ❌ NodeSelector
   - 原因：内部直接调用 edge-apiserver API

2. **包含业务逻辑的组件**
   - ❌ WorkloadsList（包含工作负载列表逻辑）
   - ❌ MetricCard（如果依赖监控 API）
   - 原因：与 Edge Platform 业务强耦合

3. **依赖 Kubb 生成类型的组件**
   - ❌ 任何引用 `src/gen/*` 类型的组件
   - 原因：Kubb 生成类型与 OpenAPI 紧耦合

## 依赖规则

### 允许的依赖

```json
{
  "dependencies": {
    "@radix-ui/*": "*",
    "lucide-react": "*",
    "class-variance-authority": "*",
    "clsx": "*",
    "tailwind-merge": "*"
  }
}
```

### 禁止的依赖

- ❌ `@tanstack/react-query` - 应由使用者注入
- ❌ 任何 Kubb 生成的 API client
- ❌ `src/gen/*` 中的类型

## 类型处理原则

### 原则
组件 props 中不直接使用 K8s API 类型，需要在 packages/ui/src/types 中重新定义泛型。

### 示例

❌ **错误**：直接使用 K8s 类型
```tsx
import { Namespace } from '@/gen/types'

interface Props {
  namespace: Namespace  // ❌ 依赖外部类型
}
```

✅ **正确**：定义泛型接口
```tsx
interface Resource {
  metadata: {
    name: string
    labels?: Record<string, string>
  }
}

interface Props<T extends Resource> {
  resource: T
  render: (item: T) => ReactNode
}
```

## 提取流程

1. **分析依赖**
   ```bash
   # 在 edge-console 中分析组件依赖
   grep -r "import.*@/gen" src/components/XXX.tsx
   ```

2. **提取组件**
   - 复制组件文件到 `packages/ui/src/components/`
   - 移除 edge-specific 依赖
   - 添加泛型类型约束

3. **编写 Story**
   - 在组件同目录创建 `*.stories.tsx`
   - 覆盖主要使用场景

4. **更新导出**
   - 在 `packages/ui/src/index.ts` 添加导出

5. **测试验证**
   ```bash
   # 在 edge-console 中替换引用测试
   pnpm build
   ```

## 示例：LabelEditor 提取

### 原始组件（edge-console）
```tsx
// src/components/dialogs/EditLabelsDialog.tsx
import { updateClusterResource } from '@/gen/api'  // ❌ 依赖

export function EditLabelsDialog({ clusterId, ... }) {
  const handleSave = async () => {
    await updateClusterResource({ ... })  // ❌ 业务逻辑
  }
}
```

### 提取后（@riseaicloud/ui）
```tsx
// src/components/LabelEditor.tsx
interface LabelEditorProps<T> {
  value: Array<{ key: string; value: string }>
  onChange: (value: Array<{ key: string; value: string }>) => void
}

export function LabelEditor<T>({ value, onChange }: LabelEditorProps<T>) {
  // 纯 UI 逻辑：键值对的增删改
}
```

### 在 edge-console 中使用
```tsx
import { LabelEditor } from '@riseaicloud/ui'
import { updateClusterResource } from '@/gen/api'

export function EditLabelsDialog({ clusterId, ... }) {
  const handleSave = async (labels) => {
    await updateClusterResource({ ... })  // ✅ 业务逻辑留在这里
  }

  return <LabelEditor value={labels} onChange={handleChange} />
}
```
