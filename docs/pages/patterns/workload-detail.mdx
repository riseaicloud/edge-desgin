# Workload Detail Pattern

工作负载详情页面的标准组合模式，覆盖 Deployment、StatefulSet、DaemonSet 等工作负载类型。

## 布局结构

```
┌─────────────────────────────────────────────────────────┐
│  ResourceNameDescription  │  ReplicaAdjustmentCard      │  ← 信息头
├─────────────────────────────────────────────────────────┤
│  KPICard × 4  (CPU / 内存 / Pod数 / 重启次数)           │  ← KPI 行
├─────────────────────────────────────────────────────────┤
│  ResourceChart (CPU 用率)  │  ResourceChart (内存用率)   │  ← 监控图表
├─────────────────────────────────────────────────────────┤
│  ContainerStatus  (容器状态)                            │  ← 容器状态
├─────────────────────────────────────────────────────────┤
│  LabelsDisplay              AnnotationsDisplay          │  ← 元数据
└─────────────────────────────────────────────────────────┘
```

## 组件清单

| 区域 | 组件 | 来源 |
|------|------|------|
| 信息头 | [`ResourceNameDescription`](/domain/resource-display) | `@riseaicloud/components` |
| 副本控制 | [`ReplicaAdjustmentCard`](/domain/resource-display/replica-adjustment) | `@riseaicloud/components` |

| 标签 | [`LabelsDisplay`](/domain/metadata) | `@riseaicloud/components` |

## 组合代码示例

```tsx
import { useState } from 'react'
import { KPICard, ResourceChart } from '@riseaicloud/ui'
import {
  ResourceNameDescription,
  ContainerStatus,
  ReplicaAdjustmentCard,
  ReplicaConfirmationDialog,
  LabelsDisplay,
  AnnotationsDisplay,
} from '@riseaicloud/components'
import { Cpu, MemoryStick, Box, RefreshCw } from 'lucide-react'

export function WorkloadDetailPage({ workload }) {
  const [desiredReplicas, setDesiredReplicas] = useState(workload.spec.replicas)
  const [confirmOpen, setConfirmOpen] = useState(false)
  const [pendingReplicas, setPendingReplicas] = useState(desiredReplicas)

  function handleReplicaChange(n: number) {
    setPendingReplicas(n)
    setConfirmOpen(true)
  }

  async function handleConfirm() {
    await scaleWorkload(workload.name, pendingReplicas)
    setDesiredReplicas(pendingReplicas)
    setConfirmOpen(false)
  }

  return (
    <div className="space-y-6 p-6">
      {/* 信息头 */}
      <div className="flex items-center justify-between gap-4">
        <ResourceNameDescription
          name={workload.name}
          alias={workload.alias}
          description={workload.description}
        />
        <ReplicaAdjustmentCard
          currentReplicas={workload.status.readyReplicas}
          desiredReplicas={desiredReplicas}
          onReplicaChange={handleReplicaChange}
        />
      </div>

      {/* KPI 行 */}
      <div className="grid grid-cols-2 lg:grid-cols-4 gap-3">
        <KPICard icon={Cpu}       title="CPU 使用率"  value={workload.cpuPercent}    unit="%" />
        <KPICard icon={MemoryStick} title="内存使用率" value={workload.memPercent}  unit="%" />
        <KPICard icon={Box}       title="Pod 数量"   value={workload.status.readyReplicas} unit="个" />
        <KPICard icon={RefreshCw} title="重启次数"   value={workload.restartCount}  unit="次" />
      </div>

      {/* 监控图表 */}
      <div className="grid grid-cols-1 lg:grid-cols-2 gap-4">
        <ResourceChart
          title="CPU 使用率"
          percentage={workload.cpuPercent}
          timeSeries={workload.cpuSeries}
          color="#3b82f6"
          unit="%"
          currentValue={`${workload.cpu.used} / ${workload.cpu.limit} cores`}
        />
        <ResourceChart
          title="内存使用率"
          percentage={workload.memPercent}
          timeSeries={workload.memSeries}
          color="#8b5cf6"
          unit="%"
          currentValue={`${workload.mem.used} / ${workload.mem.limit} GB`}
        />
      </div>

      {/* 容器状态 */}
      <ContainerStatus containers={workload.podStatus} />

      {/* 元数据 */}
      <div className="grid grid-cols-1 lg:grid-cols-2 gap-6">
        <div className="space-y-2">
          <h4 className="text-sm font-medium text-gray-700">标签 (Labels)</h4>
          <LabelsDisplay labels={workload.metadata.labels} />
        </div>
        <div className="space-y-2">
          <h4 className="text-sm font-medium text-gray-700">注解 (Annotations)</h4>
          <AnnotationsDisplay annotations={workload.metadata.annotations} />
        </div>
      </div>

      {/* 副本确认弹框 */}
      <ReplicaConfirmationDialog
        open={confirmOpen}
        onOpenChange={setConfirmOpen}
        currentReplicas={desiredReplicas}
        targetReplicas={pendingReplicas}
        onConfirm={handleConfirm}
        onCancel={() => setConfirmOpen(false)}
      />
    </div>
  )
}
```

## 布局规则

- 信息头区使用 `flex items-center justify-between`，副本卡固定右侧，与名称垂直居中对齐。
- DaemonSet 无需 `ReplicaAdjustmentCard`（不可调整副本数）。
- KPI 行移动端 2 列，桌面端 4 列。
- 图表区移动端单列，桌面端双列（`grid-cols-1 lg:grid-cols-2`）。
- `ContainerStatus` 展示最近 Pod 的容器运行状态，无 Pod 时渲染空状态。
- 元数据区两列并排，无标签/注解时显示对应 `emptyMessage`。
