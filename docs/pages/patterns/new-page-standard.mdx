# 新页面开发标准

所有新页面必须遵循 **@riseaicloud/ui + @riseaicloud/components 组合模式**。本文档定义标准架构、代码规范，并提供一个可复制的示范实现。

## 架构模型

新页面开发遵循严格的四层分工：

```
┌─────────────────────────────────────────────────────────┐
│  页面层（Page Layer）                                    │
│  组合 hooks + 组件，处理路由参数、页面级状态             │
├─────────────────────────────────────────────────────────┤
│  Layer 3: @riseaicloud/components（业务领域组件）               │
│  知道 Edge Platform 概念，接受业务数据作为 props         │
│  ClusterInfoCard, NodeCard, StatCard, MonitoringChart   │
├─────────────────────────────────────────────────────────┤
│  Layer 2: @riseaicloud/ui（纯 UI 组件）                        │
│  无业务逻辑，可在任何项目中使用                          │
│  Button, Card, Table, Dialog, EmptyState, Spinner       │
├─────────────────────────────────────────────────────────┤
│  Layer 1: @riseaicloud/tokens（设计基础）                      │
│  CSS 变量，Tailwind preset，主题系统                    │
└─────────────────────────────────────────────────────────┘
         ↑ 数据通过 props 向下传递
@riseaicloud/hooks — 数据获取层（横切关注点）
useClusters, useNamespaces, useWorkspaces, useNodeGroups
```

**关键约束：**
- @riseaicloud/components 不调用 API——数据由页面层通过 props 注入
- @riseaicloud/ui 不依赖任何业务类型——所有 props 用泛型或原始类型表达
- hooks 层只做数据获取，不包含 UI 逻辑

---

## 开发规范

| 规则 | 说明 |
|------|------|
| ✅ 使用 `@riseaicloud/hooks` 的 hooks 获取数据 | `useClusters`, `useNamespaces` 等 |
| ✅ 将数据通过 props 传入 `@riseaicloud/components` | 组件纯展示，不自己取数 |
| ✅ 使用 `@riseaicloud/ui` 处理加载/错误/空状态 | `Spinner`, `EmptyState`, `Alert` |
| ✅ 将 API 配置统一管理，不在组件内硬编码 URL | 通过 context 或页面层传入 |
| ❌ 禁止在 `@riseaicloud/components` 中调用 `fetch` 或 React Query | 违反分层原则 |
| ❌ 禁止在 `@riseaicloud/ui` 中引用业务类型（如 K8s Resource） | 组件必须保持通用 |
| ❌ 禁止在页面中直接调用 Kubb 生成的 API（绕过 hooks 层） | 统一经由 hooks 层 |

## 示范页面：Namespace 列表

下面是一个完整的示范实现，展示命名空间列表页面的标准写法。

### 目录结构

```
app/(workspace)/namespaces/
├── page.tsx              ← 页面入口，负责数据获取和组合
└── namespace-list.tsx    ← 纯展示组件（可选拆分）
```

### 第一步：数据获取（Hooks 层）

使用 `@riseaicloud/hooks` 的 `useNamespaces` hook 获取数据，**不在组件内直接调 fetch**：

```tsx
// app/(workspace)/namespaces/page.tsx
"use client"

import { useNamespaces } from '@riseaicloud/hooks'
import type { ApiClientConfig } from '@riseaicloud/hooks'

// API 配置从应用上下文获取（示例）
const apiConfig: ApiClientConfig = {
  baseUrl: process.env.NEXT_PUBLIC_API_URL ?? '',
  token: getAuthToken(), // 从 auth context 获取
}

export default function NamespacesPage() {
  const { clusterId } = useParams()

  const { data: namespaces, loading, error, refetch } = useNamespaces(
    apiConfig,
    { clusterId: clusterId as string }
  )

  // 将数据传给纯展示层
  return (
    <NamespaceListView
      namespaces={namespaces ?? []}
      loading={loading}
      error={error}
      onRefresh={refetch}
    />
  )
}
```

### 第二步：Layer 2 + Layer 3 组合展示

在展示层使用 `@riseaicloud/ui`（纯 UI）和 `@riseaicloud/components`（业务组件）：

```tsx
// app/(workspace)/namespaces/namespace-list.tsx
"use client"

import { Button, Card, CardContent, CardHeader, CardTitle, EmptyState, Spinner, Alert, AlertDescription } from '@riseaicloud/ui'
import { NamespaceStatsCards } from '@riseaicloud/components'
import type { Namespace } from '@riseaicloud/hooks'
import { Plus, RefreshCw } from 'lucide-react'

interface NamespaceListViewProps {
  namespaces: Namespace[]
  loading: boolean
  error: Error | null
  onRefresh: () => void
}

export function NamespaceListView({
  namespaces,
  loading,
  error,
  onRefresh,
}: NamespaceListViewProps) {
  return (
    <div className="flex flex-col gap-6 p-6">
      {/* 页面头部 */}
      <div className="flex items-center justify-between">
        <h1 className="text-2xl font-semibold tracking-tight">命名空间</h1>
        <div className="flex items-center gap-2">
          <Button
            variant="outline"
            size="sm"
            onClick={onRefresh}
            disabled={loading}
          >
            <RefreshCw className="mr-2 h-4 w-4" />
            刷新
          </Button>
          <Button size="sm">
            <Plus className="mr-2 h-4 w-4" />
            创建命名空间
          </Button>
        </div>
      </div>

      {/* 错误状态（Layer 2: @riseaicloud/ui） */}
      {error && (
        <Alert variant="destructive">
          <AlertDescription>
            加载失败：{error.message}
            <Button variant="link" size="sm" onClick={onRefresh} className="ml-2 p-0 h-auto">
              重试
            </Button>
          </AlertDescription>
        </Alert>
      )}

      {/* 加载状态（Layer 2: @riseaicloud/ui） */}
      {loading && !namespaces.length && (
        <div className="flex justify-center py-16">
          <Spinner size="lg" />
        </div>
      )}

      {/* 统计卡片（Layer 3: @riseaicloud/components） */}
      {namespaces.length > 0 && (
        <NamespaceStatsCards
          namespaces={namespaces.map(ns => ({
            name: ns.name,
            id: ns.id,
          }))}
        />
      )}

      {/* 命名空间列表 */}
      {!loading && !error && namespaces.length === 0 ? (
        // 空状态（Layer 2: @riseaicloud/ui）
        <EmptyState
          title="暂无命名空间"
          description="在此集群中还没有命名空间，点击「创建命名空间」开始。"
          action={
            <Button size="sm">
              <Plus className="mr-2 h-4 w-4" />
              创建命名空间
            </Button>
          }
        />
      ) : (
        // 列表卡片（Layer 2: @riseaicloud/ui 布局 + Layer 3 内容）
        <div className="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3 gap-4">
          {namespaces.map(ns => (
            <NamespaceCard key={ns.id} namespace={ns} />
          ))}
        </div>
      )}
    </div>
  )
}
```

### 第三步：单项展示组件

单个命名空间卡片使用 Layer 2 的 `Card` + `Badge` 组合：

```tsx
import { Card, CardContent, CardHeader, Badge } from '@riseaicloud/ui'
import { LabelsDisplay } from '@riseaicloud/components'
import type { Namespace } from '@riseaicloud/hooks'

function NamespaceCard({ namespace }: { namespace: Namespace }) {
  return (
    <Card className="hover:shadow-md transition-shadow">
      <CardHeader className="pb-2">
        <div className="flex items-start justify-between">
          <span className="font-medium text-sm">{namespace.name}</span>
          <Badge variant="secondary">Active</Badge>
        </div>
      </CardHeader>
      <CardContent>
        {namespace.labels && Object.keys(namespace.labels).length > 0 && (
          <LabelsDisplay
            labels={namespace.labels}
            maxVisible={3}
          />
        )}
      </CardContent>
    </Card>
  )
}
```

## 完整组合示意图

```
NamespacesPage (page.tsx)
│
├── useNamespaces()  ─── @riseaicloud/hooks ──→ API 请求
│     └── { data, loading, error, refetch }
│
└── <NamespaceListView>  ─── 页面层组合
      ├── <Alert>          ─── @riseaicloud/ui (Layer 2)  错误提示
      ├── <Spinner>        ─── @riseaicloud/ui (Layer 2)  加载中
      ├── <NamespaceStatsCards>  ─── @riseaicloud/components (Layer 3)
      ├── <EmptyState>     ─── @riseaicloud/ui (Layer 2)  空状态
      └── <NamespaceCard>  ─── 本地组合
            ├── <Card>         ─── @riseaicloud/ui (Layer 2)
            ├── <Badge>        ─── @riseaicloud/ui (Layer 2)
            └── <LabelsDisplay> ── @riseaicloud/components (Layer 3)
```

**数据流向（单向）：**
```
API → useNamespaces() → NamespacesPage → props → NamespaceListView → props → 各子组件
```

## 反面示例（禁止）

### ❌ 在 @riseaicloud/components 内直接调用 API

```tsx
// 错误：组件内部调用 fetch
export function NamespaceSelector({ clusterId }: { clusterId: string }) {
  const [namespaces, setNamespaces] = useState([])

  useEffect(() => {
    fetch(`/api/namespaces?clusterId=${clusterId}`)
      .then(r => r.json())
      .then(setNamespaces)
  }, [clusterId])

  // ...
}
```

✅ **正确写法**：数据由外部注入：

```tsx
// @riseaicloud/components 中的 NamespaceSelector 接受 props 而非自己取数
export function NamespaceSelector({
  namespaces,    // 外部传入的数据
  loading,
  onSearch,
  ...
}: NamespaceSelectorProps) {
  // 纯 UI 渲染
}
```

### ❌ 在页面中直接用 Kubb 生成的 API

```tsx
// 错误：绕过 hooks 层，直接调用 Kubb client
import { listNamespaces } from '@/gen/api'

export default function Page() {
  const { data } = useQuery({
    queryFn: () => listNamespaces({ clusterId }),
  })
}
```

✅ **正确写法**：通过 `@riseaicloud/hooks` 统一封装：

```tsx
import { useNamespaces } from '@riseaicloud/hooks'

export default function Page() {
  const { data } = useNamespaces(apiConfig, { clusterId })
}
```

### ❌ 混合 UI 和数据逻辑

```tsx
// 错误：UI 组件里包含业务逻辑判断
function NamespaceCard({ namespaceId }: { namespaceId: string }) {
  // 组件自己查询，违反单一职责
  const data = useNamespaceDetail(namespaceId)
  const canDelete = checkPermission(data?.labels)
  // ...
}
```

✅ **正确写法**：将数据和权限判断提升到页面层，组件只接受最终结果：

```tsx
function NamespaceCard({ namespace, canDelete }: NamespaceCardProps) {
  // 纯展示，无数据依赖
}
```

## 快速检查清单

新页面提交前检查：

- [ ] 数据获取统一使用 `@riseaicloud/hooks` 中的 hooks
- [ ] 业务组件（@riseaicloud/components）只接受 props，不调用 API
- [ ] 加载、错误、空状态均有处理（使用 @riseaicloud/ui 的 Spinner / Alert / EmptyState）
- [ ] 没有硬编码 API URL 或 token
- [ ] TypeScript 类型正确，无 `any`
- [ ] 页面不超过 200 行；超过时拆分为子组件文件
